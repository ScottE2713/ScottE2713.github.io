Milestone 3: Enhancement 2 - Algorithms and Data Structure

Artifact Description

The artifact selected for this category’s (Algorithms and Data Structures) enhancement was AdvisingProgram.cpp. This was a C++ console app that I created when I took CS300. The program is basically a course catalog. It reads data from a CSV file, stores the data in a Binary Search Tree (BST), then allows you to list all courses or print certain courses to get information about those courses.

Justification and Enhancements

I decided that this artifact was a good choice because it has the potential to serve as a strong foundation for demonstrating my knowledge of data structures and algorithms. The enhancement that I implemented was a prefix-based search feature. This lets the user retrieve all the courses that have a specific course number prefix (for example, CS). To implement this, I needed to update the binary search tree with a custom recursive traversal function that collected all the nodes that match the user-specified prefix.

As part of this enhancement, I included some input validation and normalization like trimming, removing hyphens (which improves the UX), and converting all course numbers to uppercase. With this enhancement, I improved upon the original program’s functionality while demonstrating my understanding of traversal logic, string processing, and defensive programming.

Reflection on Enhancement Process

By enhancing this artifact, I have improved on my understanding of tree traversal and algorithm design. I needed to be careful and pay close attention to input conditions, edge cases, and runtime behavior when implementing the recursive search logic here. I also tested the program using a dataset that contains 400 courses. This data set included malformed and valid inputs, to ensure accuracy and resilience.

One challenge that I faced when enhancing this artifact was that I kept running into iterator errors when trying to trim and validate malformed strings. After doing some research on how others solve this problem, I was able to overcome this by updating my utility functions and implementing better validation and formatting logic. Even though fixing this wasn’t a direct security feature, the enhancement does mitigate iterator errors. This supports foundational secure coding practices. This experience reinforced the value of modular design and defensive programming. Overall, this enhancement is a strong upgrade to this artifact that improved upon the program’s usability while also demonstrating my knowledge and skills in algorithms and data structure.
