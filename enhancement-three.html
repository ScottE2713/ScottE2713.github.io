Milestone 4: Enhancement Three – Database Integration

Artifact Description

The artifact that I picked for the third enhancement is AdvisingProgram.cpp. It is a C++ console application that serves as an advising program to track offered courses along with their prerequisite courses. The program was originally created for CS-300 and was designed to load and display course data from a flat file using a Binary Search Tree (BST).

Initially, the program only supported reading from a static file, with no ability to modify data through the application. I enhanced the program by integrating a SQLite database, which enables full Create, Read, Update, and Delete (CRUD) functionality with persistent storage.

Justification and Enhancements

I selected this artifact because it demonstrates my ability to extend an existing C++ application by integrating secure, database-driven functionality. These enhancements showcase my skills in software extension, SQL integration, and secure query practices.

I created a DatabaseAdapter class to encapsulate all SQLite operations using reusable, parameterized methods. The original file-loading feature was preserved to allow bulk importing of new data. I added menu options for adding, editing, deleting, and viewing records from the database, and ensured that the BST remained synchronized with the database for in-memory searching. All queries are securely handled to prevent SQL injection.

Outcome Alignment

This enhancement supports multiple CS499 outcomes. It demonstrates my ability to design and implement computing solutions using appropriate tools by replacing the flat file with a relational SQLite database. It also highlights my use of innovative techniques through parameterized queries and safe adapter design. Finally, it reflects my secure development mindset by applying validation, safe memory use, and SQL injection prevention.

Reflection on Enhancement Process

Through this enhancement, I gained a deeper understanding of SQL integration and database-aware architecture. I carefully planned the transition from a file-based system to a persistent model, while preserving the in-memory BST structure introduced in earlier enhancements.

One challenge was ensuring that the new database functionality did not interfere with the existing BST logic. I also deepened my knowledge of secure query logic using sqlite3_prepare_v2 and sqlite3_bind_text, reinforcing best practices to avoid raw SQL execution.

In conclusion, this enhancement helped me deliver a more scalable, secure, and professional-quality C++ application that effectively uses a relational database.
